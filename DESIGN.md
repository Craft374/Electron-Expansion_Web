## 0. 밸런스 목표 (최우선 기준)

| 구간 | 목표 시간 | 비고 |
|---|---|---|
| 게임 시작 → 원소 10번대(네온~) | **2시간 이내** | 극초반은 빠르게 통과 |
| 원소 10번 → 26번(철) | 남은 ~20시간, **1시간당 약 1원소** | 천천히, 대신 합성으로 지루함 방지 |
| 철 → 첫 항성 | 약 1일(24h) 부근 | 단, **첫 리셋은 항성보다 앞이어야 함** (§4 후보) |

핵심 원칙: **원소 진행은 느리지만, 그 사이를 합성(§3)·도전과제·자동화로 계속 할 게 있게** 채운다.

## 1. 현재 구현 상태 (2026-07, v2.2.0) 🔧

- **입자**: 전자→양성자→중성자 생성기 + 변환기(입자→E). 수동/자동 변환.
- **엔트로피(E)**: 메인 화폐.
- **원소**: 수소~철(26종). 이전 원소로부터 연쇄 생산, 보유량만큼 E 생산. 업그레이드 4종(융합 강화·엔트로피 응축·수소 선속·입자 가속) + 특수 연구(원소 소모 1회성).
- **동위원소**: 중성자 소모 보조 아이템.
- **도전과제(업적)**: A~E줄, 줄 완성 보상. 우측 상단 토스트.
- **엔트로피 희생**: 붕소 이후. 초당 E를 잠시 0으로, 복구 100%에서만 가능. 중성자 생성기에 아주 약한 영구 배율.
- **자동화**: 생성기·변환기·원소 업그레이드 자동 구매. 딜레이 60초→0초(5초 단위), 개별/전체 on-off, 개별/전체 최대 단축.
- **항성**: 철 모아 점화 → E 증폭 + 원소 직접 생산. 온도로 원소 편향. 최대 Lv365.
- **행성**: 티어1 랜덤 → 티어2 태양계행성 → 티어3 커스텀 → 티어4 특이행성. **태양 레벨 50당 한 티어씩 공개**.
- **태양계**: 태양 Lv365 + 행성 9개 → 통합(현재 버전 끝).

### 알려진 밸런스 문제 (수정 대상)
- **원소 구조 역전 필요** → §2
- **항성 철 요구량 과다**(1e100) → §5
- **첫 리셋이 항성뿐이라 너무 늦음** → §4

## 2. ⏳ 원소 구조 역전 (H가 제일 많고, 원자번호↑ = 적게)

**현재(문제)**: 연쇄 생산이 누적되어 상위 원소가 더 많아짐(수소 최소, 바나듐 최대). 직관과 반대.

**목표**: 수소가 제일 많고 원자번호가 커질수록 적어지는 **감소 계단** 구조.

### 제안 모델 (Claude 구현안)
원소 N을 독립 누적하지 않고 **상위일수록 생산 계수를 급감**시켜 항상 `원소[N] < 원소[N-1]`이 유지되게 한다.

- 원소 N 생산 = `원소[N-1] × cascade × decayᴺ` (decay < 1, 예: 0.15)
  → 상위 원소는 하위의 일정 비율에서 평형 → 자연히 계단형 감소.
- E 생산은 반대로 상위일수록 개당 E를 크게(`eGrow`) 유지 → "적지만 값진 상위 원소" 밸런스.
- **원소 최대 개수 `1.8e308`(Number 한계)로 캡, 항성 획득 이후 캡 해제.**
  - 구현: `tick()`에서 `if (!state.star) elements[i] = min(elements[i], 1.8e308)`.

### 자동화 비용도 함께 반전
- 초반 단축 = 원자번호 작은 원소(H, He…), 후반 단축 = 큰 원소(…Fe) 요구.
  - 현재는 임시로 "최신 원소" 기준. 역전 모델 확정 후 `autoStepCost`를 원자번호 진행형으로 교체.

✏️ 결정: decay 값(감소 기울기), 원소별 최대 목표량 곡선.

## 3. ⏳ 합성 시스템 (신규 탭) — "지루함 방지 핵심 콘텐츠"

**해금**: 수소 연구 직후 `합성` 탭 등장.
**컨셉**: 원소들을 조합해 **화합물**을 연구/제작 → 구성 원소에 생산 배율. 레벨업 가능.

### 시스템 규칙 (구현안)
- 각 화합물: `연구(1회 해금)` → 이후 `레벨업`(원소 소모)으로 배율 강화.
- 배율은 **구성 원소에만** 적용(예: 물 = 수소·산소 ×배율).
- 원소가 너무 겹치지 않게 분산 배치(아래 표는 대표 원소 기준).
- **네온 등 비활성 기체**는 일반 화합물 불가 → "특수 반응"으로 이론상 가능한 것만(엑시머·클라스레이트·불소화물).
- 페이싱: 원소 하나 해금할 때마다 새 화합물 2~4개가 열리도록 요구 원소를 계단식으로.

### 화합물 목록 초안 (약 40종) — ✏️ 제목/효과 자유 수정
> 형식: **화학식 (이름)** — 필요 원소 → 효과

## 4. ✏️ 첫 리셋(프레스티지) — 항성보다 앞선 레이어 후보

> 문제: 지금은 첫 리셋이 항성(≈1일)뿐이라 너무 늦음. **더 이른 리셋 레이어**가 필요.
> 아래 후보 중 고르거나 조합하세요. Claude가 선택안을 구현합니다.

### 후보 A — "핵 압축(Compression)" · 매우 이른 리셋 (붕소~탄소, ~30분)
- 리셋 대상: 입자·생성기·변환기·E (원소는 유지)
- 보상: **압축 포인트(CP)** → 입자 생산 영구 배율 / 변환 효율 업그레이드
- 루프: 수 분~십수 분. 안티매터의 "무한(Infinity)"에 해당하는 첫 프레스티지.
- 장점: 아주 이른 반복 재미. 단점: 레이어 하나 신규 UI 필요.

### 후보 B — "핵종 붕괴(Decay)" · 중반 리셋 (네온~규소, ~2–4시간)
- 리셋 대상: 원소 전체 + 위 전부
- 보상: **핵종 포인트(NP)** → 원소 생산 전역 배율 / 연구 비용 할인 / 시작 시 원소 소량 지급
- 루프: 수십 분~1시간. 안티매터의 "이터니티"에 해당.

### 후보 C — 2단 구조 (A + 항성)
- 이른 리셋 = 후보 A(입자층), 큰 리셋 = 항성(§5). 항성은 그대로 유지하되 A를 앞에 추가.

### 후보 D — 리셋 없이 "봉인 해제"형
- 리셋 대신, 일정 목표 달성 시 영구 버프를 주는 마일스톤 레이어(반쯤 프레스티지). 손실 없음.

## 5. ⏳ 항성 = 리셋/프레스티지 시스템 (사용자 요청 §11)

**철 밸런스 먼저**: 현재 항성 레시피 철 1e100 요구가 과다 → 획득 곡선에 맞게 하향(✏️ Claude 조절).

### 항성 붕괴 루프 (구현안)
1. 첫 항성 도달: 약 **1일**. 항성 생성 시 **모든 업그레이드 초기화**(원소·입자·자동화·희생 등), 항성 보너스만 유지.
2. 항성 보유 상태로 **다시 항성 조건 도달** → 기존 항성을 **붕괴**시키고 재생성.
   - 재도달 시간: 1회차 후 약 **6시간** → 붕괴할수록 점점 단축.
   - 항성 자체가 **다음 붕괴를 더 빠르게** 해주는 보너스 보유(누적).
3. 붕괴까지 걸리는 시간이 **1초 이하**가 되면 → **영구 항성 시스템** 가동.
   - 이후 붕괴 불가. 대신 **항성 업그레이드** 활성화.

### 항성 업그레이드 종류 (💭 다양하게 — 초안)
- E 생산 배율 / 원소 직접 생산량 / 온도 상한·효율
- 자동화 딜레이 전역 감소 / 자동화 무료화
- 희생 배율 강화 / 오프라인 시간
- 원소 연구 비용 할인 / 합성 배율 강화
- 붕괴 시 시작 자원 지급(스타트 대시)
- 행성 티어 조기 공개 / 행성 보너스 강화
✏️ 항성 업그레이드 트리 구조(선형/분기/격자), 각 비용 화폐(항성 포인트?).

✏️ 결정: 붕괴 시간 단축 곡선(6h→…→1s), 영구화 조건, 항성 포인트 획득식.

## 6. 💭 행성 디테일 강화 (사용자 요청)

> 지금은 "막 깔아두는" 느낌. 행성 하나하나가 **의미**를 갖게.

아이디어 후보:
- **행성별 고유 이벤트/미션**: 예) 화성=테라포밍 단계별 목표, 목성=대적점 폭풍 관리.
- **행성 성장**: 행성도 항성처럼 레벨/파라미터(질량·대기·궤도)를 갖고 키우기.
- **행성 시너지**: 특정 조합(내행성 4개, 가스행성 4개 등) 완성 시 세트 보너스.
- **행성 자원 채굴**: 행성이 특정 원소/특수 자원을 직접 산출, 업그레이드로 증대.
- **커스텀 행성 심화**: 대기·핵·표면을 각각 설계(현재 원소 조합보다 깊게).
