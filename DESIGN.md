# Electron Expansion — 설계 문서 (틀)

> Idea1.md / idea2.md 를 정리해서 만든 게임 설계 틀입니다.
> ✏️ 표시가 있는 곳 = 아직 미정이라 직접 채워 넣으면 되는 부분.
> 모든 숫자(가격, 속도 등)는 **임시값**이고 `js/data.js` 파일에서 전부 수정 가능합니다.

---

## 1. 게임 한 줄 요약

> 입자를 모아 원소를 만들고, 원소로 우주를 만들고, 결국 시뮬레이션을 탈출하는 증분형(방치형) 게임.

**핵심 루프**: 입자 생성 → 엔트로피(E)로 변환 → 업그레이드 구매 → 더 빠른 생산 → 상위 단계 해금 → (리셋) → 반복

---

## 2. 용어 정리 (자원)

| 자원 | 단위/표기 | 역할 |
|---|---|---|
| 엔트로피 | E | 메인 화폐. 거의 모든 업그레이드에 사용 |
| 양성자 | 개 | 기초 입자. E로 변환하거나 원소 재료로 사용 |
| 전자 | 개 | 기초 입자. 원소 재료. E 변환 효율은 일부러 나쁘게 (전자 2개 → 1E) |
| 중성자 | 개 | 수소를 변환해서 획득. 중수소/삼중수소 재료 |
| 원소 (수소 등) | 개 | 진짜 컨텐츠. 보유하면 초당 E를 자동 생산 |

✏️ 이름 바꾸기 예정 (idea2 할 일 목록): "프로톤" 등 오브젝트 이름 확정 필요.

---

## 3. 진행 단계 로드맵

### Phase 1 — 입자 시대 ✅ (구현됨)
- 클릭으로 양성자 생성 → 수동으로 E 변환 → 자동화 구매
- **양성자 생성기**: a초당 1개 생성. a는 1.0에서 시작해 업그레이드로 0.1까지 감소
- **자동 변환기** (500 E 해금): a초당 양성자 N개를 E로 자동 변환. 같은 방식으로 가속
- **전자 생성기** (250 E 해금): a초당 2개 생성. 같은 방식으로 가속
- ⏱️ 페이싱 목표: 이 단계는 "극초반 통과 구간". 원소 도달까지 **2~3시간** (1시간은 너무 짧음)

### Phase 2 — 원소 시대 ✅ (수소 구현, 나머지 뼈대만)
- **수소 연구**: 양성자 50 + 전자 100 + 75 E (1회)
- **수소 생산**: 양성자 1 + 전자 1 + 14 E → 수소 1개. 수소 1개당 **초당 10 E** 자동 생산
- **수소 → 중성자 변환**: 수소 1개 → 중성자 1개 (10 E/초 생산을 포기하는 대신 상위 원소 재료 획득)
- **중수소 연구** ✏️(비용 미정 — 현재 임시값): 생산 = 양성자1 + 전자1 + 중성자1 + 25 E → 초당 5 E
- **삼중수소 연구** ✏️(비용 미정 — 현재 임시값): 생산 = 양성자1 + 전자1 + 중성자2 + 40 E → 초당 8 E
- 💡 방향성: 여기가 진짜 컨텐츠. 유저가 할 게 제일 많고 선택지가 많은 구간으로 확장

### Phase 3 — 핵융합 (첫 번째 리셋 / 프레스티지) 🔒 (탭만 존재, 미구현)
- 핵융합 실행 시 **모든 입자/원소/E가 사라지고** 행성 제작
- ✏️ 정할 것: 핵융합 조건(원소 몇 개?), 행성이 주는 보너스(예: E 생산 배수), 리셋 후 뭐가 유지되는지

### Phase 4 — 천체 계층 (미구현)
- 행성 → 항성 → 태양계 → 은하 → 은하단 → 우주
- 각 계층이 하나의 리셋 레이어 (안티매터 디멘션의 인피니티/이터니티 같은 구조)
- ✏️ 정할 것: "다양한 루트"가 무엇인지 (계층마다 선택 분기?)

### Phase 5 — 멀티버스 (미구현, 브레인스토밍 단계)
- 멀티버스 챌린지
- 멀티버스 전용 아이템 시스템 (idea2 브레인스토밍):
  - 무조건 E를 생산하는 아이템 / 양성자+전자만으로 원소 생성 가능한 아이템
  - 멀티버스 포인트로 업그레이드, 또는 아이템 뽑기
  - 진입 전에 설명만 보고 유저가 직접 아이템을 조합해서 들어가는 방식
- ✏️ 정할 것: 아이템 목록, 획득 방식(뽑기 vs 포인트), 허블망원경/블랙홀 컨셉의 역할

### Phase 6 — 에러 (미구현)
- 멀티버스 속 에러 수집 → 에러를 모아 "에러의 멀티버스" 제작
- 에러의 멀티버스에서 시뮬레이션 흔적 발견

### Phase 7 — 엔딩 (미구현)
- 시뮬레이션 탈출 방법 탐색 → 탈출 → 엔딩

## 4. 공통 시스템 규칙

### 4-1. 속도 업그레이드 규칙 (idea2 방식, 모든 생성기 공통)
"a초당 1사이클"에서 a가 다음 순서로 감소:

```
1.0 → 0.9 → 0.8 → 0.7 → 0.6 → 0.5   (0.1씩 감소)
0.5 → 0.45 → 0.4 → 0.35 → 0.3 → 0.25 → 0.2 → 0.15 → 0.1   (0.05씩 감소)
0.1 도달 시 업그레이드 종료 (최대 초당 10사이클)
```

이 표는 `js/data.js`의 `SPEED_STEPS`에 있고, 모든 생성기가 공유합니다.

### 4-2. 변환 규칙
- 양성자 → E: **1:1** (수동 전량 변환 버튼 + 자동 변환기)
- 전자 → E: **2:1** (일부러 비효율적으로 — 전자는 원소 재료로 쓰는 게 이득)
- ✏️ 미정: E → 전자 역변환을 넣을지 (idea2 할 일 목록에 있었음)

### 4-3. 저장
- 30초마다 자동 저장 + 브라우저를 닫아도 저장 (localStorage)
- 설정 탭에서 수동 저장 / 내보내기(백업 코드) / 가져오기 / 초기화
- 오프라인 보상: 꺼둔 동안의 생산을 접속 시 지급 (최대 8시간, `data.js`에서 수정 가능)

## 5. UI 구성 원칙 (난잡하지 않게)

- **탭 방식** (안티매터 디멘션 스타일): `입자 | 원소 | 핵융합 | 설정`
- **점진 공개**: 아직 도달 못 한 컨텐츠는 탭/카드 자체를 숨기거나 🔒 잠금 표시
  - 원소 탭 → 전자 생성기 구매 후 등장
  - 핵융합 탭 → 수소 연구 후 등장
- 한 화면에 카드 몇 장만. 스크롤 최소화 (idea2: "제일 중요한 건 스크롤 시스템")
- 다크 테마, 한글 기본

## 6. 파일 구조 (어디를 고치면 뭐가 바뀌나)

```
index.html      화면 구조 (카드, 버튼, 문구)
style.css       디자인 (색, 크기, 배치)
lib/break_eternity.min.js   큰 수 라이브러리 (건드릴 필요 없음)
js/data.js      ⭐ 밸런스 수치 전부 여기 (가격, 속도, 비용) — 제일 자주 만질 파일
js/format.js    숫자 표기 (1,234 / 1.23e15 형식)
js/game.js      게임 규칙 (생산, 변환, 연구, 구매 로직)
js/save.js      저장/불러오기/오프라인 보상
js/ui.js        화면 갱신 (숫자 표시, 버튼 활성화, 탭)
js/main.js      시작점 (게임 루프 실행)
```

**실행 방법**: `index.html` 파일을 더블클릭하면 브라우저에서 바로 실행됩니다. 설치 필요 없음.

## 7. ✏️ 네가 채워 넣을 세세한 요소 목록

1. **밸런스**: `js/data.js`의 모든 가격/비용 (지금은 전부 임시값). 목표: 원소까지 2~3시간
2. **중수소/삼중수소 연구 비용**: idea1에서 `@개`로 비워둔 부분
3. **오브젝트 이름 확정**: 양성자/전자/중성자/수소... 이대로 갈지, 바꿀지
4. **E → 전자 변환**: 넣을지 말지, 넣는다면 비율
5. **핵융합 조건과 보상**: 첫 리셋 설계 (제일 중요한 다음 단계)
6. **멀티버스 아이템**: 목록, 획득 방식, 조합 규칙
7. **스토리 문구**: 각 단계 해금 시 보여줄 짧은 텍스트 (시뮬레이션 떡밥 등)
